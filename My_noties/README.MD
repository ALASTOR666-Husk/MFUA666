# 🔄 Синхронизация Git-репозиториев

Скрипт автоматизирует процесс обновления локального репозитория из удалённого источника и последующего переноса файлов в другой локальный каталог.

---

## ⚠️ Важные замечания

1.  **Проверка путей:** Перед запуском обязательно убедитесь, что переменные `SOURCE_DIR` и `DEST_DIR` в скрипте указаны верно. Некорректный путь может привести к потере или перезаписи данных.
2.  **Имена файлов:** Избегайте использования пробелов, символов кириллицы и специальных знаков в названиях файлов и директорий репозиториев.

---

## 📋 Системные требования

-   Установленная система контроля версий **Git**.
-   **Для Bash:** утилита `rsync` (по умолчанию доступна в Linux и macOS).
-   **Для PowerShell:** ОС Windows 10/11 (утилита `Robocopy` встроена в систему).

---

## ⚙️ Настройка

1.  Откройте файл скрипта (`migrate_git.sh` или `migrate_powershell.ps1`) в любом текстовом редакторе.
2.  Найдите раздел **«Настройки»**.
3.  Укажите актуальные пути к вашим папкам:
    -   `SOURCE_DIR` — путь к основному репозиторию (откуда будут загружаться обновления через `git pull`).
    -   `DEST_DIR` — путь к целевому репозиторию (куда будут скопированы файлы).

---

## 🚀 Запуск скрипта

### Вариант 1: Bash (Linux / macOS)

```bash
# 1. Делаем скрипт исполняемым
chmod +x migrate_git.sh

# 2. Запускаем синхронизацию
./migrate_git.sh
```

### Вариант 2: PowerShell (Windows)

```powershell
# 1. Разрешаем выполнение скрипта в текущей сессии
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

# 2. Запускаем синхронизацию
.\migrate_powershell.ps1
```
